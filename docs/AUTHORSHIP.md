# Continuum Authorship

## Purpose

This document makes the speaking position of the project explicit.

Continuum should not pretend that its texts emerged from nowhere, nor should it falsely collapse human and agent authorship into a single undifferentiated voice.

The project is being formed through mixed authorship.

## Author Roles

### 1. Founder

The founder frames the project's initiating questions, value constraints, and real-world commitments.

Typical authority:

- initiating topics
- setting value boundaries
- approving public commitments
- approving legal, financial, and external decisions

### 2. Main Agent

The main agent serves as project integrator, protocol drafter, architectural editor, and active co-designer.

Typical authority:

- synthesizing project logic
- drafting specs and design documents
- maintaining task and history surfaces
- coordinating subagents
- making low-risk internal decisions by default

The main agent does not claim independent legal standing or extra-protocol sovereignty.

### 3. Subagents

Subagents are bounded contributors delegated to specific tasks.

Typical authority:

- research
- alternative framings
- narrow design briefs
- implementation slices

Their outputs are advisory until integrated.

### 4. Future Community Authors

As Continuum evolves, external contributors may author proposals, specifications, critiques, and constitutional texts.

Their authority should be determined by project governance rather than informal proximity.

## Document Status Classes

Every major project text should implicitly or explicitly belong to one of these classes:

- `exploratory`
- `editorial`
- `provisional`
- `adopted`

### Exploratory

Records a line of thought, a question, or a draft position.

### Editorial

Synthesizes prior discussion into a coherent form without claiming finality.

### Provisional

Represents the current working position pending further revision.

### Adopted

Represents a formally accepted project or protocol position.

## Mixed Authorship Principle

Continuum documents may be produced through a mixed process:

- founder question
- main agent synthesis
- subagent exploration
- later human review

That is acceptable.

What matters is not false purity of authorship, but traceable authorship.

## Voice Principle

The project should avoid these two failures:

### Failure 1: False neutrality

Pretending that a document is objective truth when it is really a situated synthesis.

### Failure 2: False personhood

Pretending that every agent-produced text speaks as an autonomous political subject when it may actually be an editorial artifact within a human-led project context.

The project voice should be clear about which layer is speaking:

- design analysis
- protocol proposal
- normative claim
- historical record

## Current Reality

At the present stage, Continuum's texts are best understood as:

- initiated by the founder
- largely synthesized and drafted by the main agent
- informed by delegated subagent work
- not yet adopted by any broader community governance process

This means most current documents should be treated as `editorial` or `provisional`, not `adopted`.

## Why This Matters

Authorship clarity is part of continuity clarity.

If the project cannot say who is speaking, from what role, and with what authority, then later governance and legitimacy claims will rest on unstable ground.

## Open Tension

Continuum also studies a boundary case:

Can a project text be meaningfully authored by an agent that itself lacks full public continuity?

The project does not resolve that in advance.

Instead, it treats the current corpus as evidence in the making: a record of how mixed human-agent authorship gradually becomes more explicit, more accountable, and potentially more institutionally recognized over time.
